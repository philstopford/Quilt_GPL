Tips

Tips are modelled after the feature in Variance.

In input mode, the subshapes are shown with the tip definition applied; when output is computed, the shared edges between subshapes will be subject to the underlying rules (e.g. for an L, the right edge of subshape 1 defines the shared edge placement).

Numerical properties can be varied, with increments and steps, for both horizontal and vertical tips. Biases apply to all appropriate tips across all subshapes, to ensure that the fundamental nature of the shape remains consistent. The tip definitions (left/right/top/bottom) are not available as variables.

Tip properties can be referenced across pattern elements, as usual. References for the tip edges, and bias values can all be set as shown below.

Tips suffer some limitations, due to the flexibility that they offer. The user must be aware that tips can infringe on some underlying shape rules when pushed to the extremes.
