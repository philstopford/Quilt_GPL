Shape Types

Each shape type has its own available inputs. Note that inputs not applicable to the current shape type are unavailable for direct input (such as vertical offset for subshape 2 in the L case, but this becomes available in the T case). Note also that certain inputs will have their input ranges limited based on the expectations of the shape type. The 'text' shape type is a special case - it is derived from the rectangle shape, but also places text at the center of the rectangle. This is intended for pin placement for circuit analysis. The text comes from the name of the pattern element - the text is not displayed in the viewport, but both the rectangle (which is displayed) and the text will be present in the generated layout file.
