Quilt (5.0)

Phil Stopford

Introduction

Quilt is a production-proven, high performance pattern generator that writes to hierarchical GDSII and OASIS files. The tool also works with the DOE system in Variance (3.5.1 or later). It is available on Windows, macOS and Linux, running on top of NET 7

Features

  • Pattern elements based on various shapes (box, L, T, U, X, S, text, bounding box).
    • Text shape for pattern - rectangular primtive that results in a text element in the exported layout.
    • Pattern element from bounding box of pattern.
    • Arrays can be constructed from the pattern element.
      • Array counts can be independently varied.
      • Array spacings can be independently varied.
    • Interlayer subshape-referenced dimensions.
    • Tip definitions.
      • Horizontal and vertical tips can be defined.
      • Tip extensions are pattern properties, allow variations to be assessed.
      • Tip properties can be referenced across elements.
  • >Patterns can now be defined using any cell in a layout file (GDSII or OASIS, including GZIP-compressed and strict OASIS).
    • Rectangular decomposition now runs 5x faster due to library improvements.
    • This can now be cancelled in case the decomposition is taking longer than desired.
    • Mapping to internal primitives (box, L, T, U, X, S) with transforms is attempted.
    • Complex orthogonal geometry will be decomposed to rectangles, with hierarchical, relative transformations applied to ease variation set-up.
      • Greatly improved performance due to new Boolean library.
      • Decomposed geometry will be reconstituted during export, to provide minimal output.
      • Orientation of decomposition rectangles is configurable.
      • Decomposition reduces to a minimum number of rectangles.
      • Decomposition now handles loops, using the keyholer.
      • Decomposition of multi-islands and reconstitution on-export.
    • Complex primitive type used for cases that cannot be mapped to the primitives or decomposed.
      • User interface now contains the coordinate list in a collapsed expander section, to avoid making the user interface unwieldy.
  • Pattern elements can be copied, pasted, removed and duplicated from a context menu.
  • Pattern elements can be renamed.
  • Names of pattern elements can be used to define the layer and datatype used on-export.
  • Pattern elements can be merged if desired during export.
  • Full range of variations for pattern generator: shape characteristics and positioning.
    • Interlayer subshape-referenced dimensional variations.
    • Interlayer subshape-relative positioning.
    • Rotation, including relative rotation. Relative rotation can use reference pivot point.
      • Rotation pivot point choice now includes 'world origin', alongside 'self' and reference.
    • Array rotation, including relative rotation. Relative rotation can use reference pivot point.
      • Rotation pivot point choice now includes 'world origin', alongside 'self' and reference.
  • Pattern extents can be displayed to ease understanding of the quilt construction.
  • Flip (horizontal, vertical, also aligned in either or both of X and Y) support.
  • Text annotation (pins).
  • Reference elements can be identified using 'Find' buttons and context menus across the user interface, easing workflow for reviewing and editing these relationships.
  • High performance, multi-threaded pattern generation engine, supporting arbitrary pattern definitions and automatic duplicate removal. 225,000 patterns takes less than 10 seconds.
    • Relative transforms are very fast, due to a sophisticated transformation pipeline.
  • Loading of patterns with many elements (e.g. from decomposed geometry) is very fast, due to multithreading in the loader.
  • Different colors can be assigned to geometry from the selected / unselected pattern elements.
  • Export to hierarchical GDSII and OASIS. The export process is extremely fast, taking advantage of multithreading and optimized design.
    • Resolved a defect with OASIS output for certain geometry.
    • GZIP-compressed output files are supported.
  • Auto-configuration of Variance DOE.
  • Multi-platform : Windows, macOS, Linux. No operational or functional difference across platforms.
  • Modern viewport automatically detects and utilizes optimal framework for platform (Vulkan, OpenGL, Direct3D, Metal).
    • NOTE : nVidia driver problems prevent OpenGL under Linux. Vulkan is fine.
    • Middle (or Ctrl-Left) click selection of elements in the viewport.
    • Customization of background and axis colors.
  • Runs under NET 7 on all platforms, with standalone packages that include the runtime.
